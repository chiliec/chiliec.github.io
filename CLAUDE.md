## Project

Personal site / blog for Vladimir Babin, served at https://babin.info (custom domain on GitHub Pages, source repo `chiliec.github.io`). Built with **Hexo 8** (static site generator, Node.js). Content language is Russian. The default theme `landscape` lives in `themes/landscape/` and is checked into the repo.

## Commands

```bash
npm run server   # hexo server ‚Äî local dev at http://localhost:4000
npm run build    # hexo generate ‚Äî outputs to ./public
npm run clean    # hexo clean ‚Äî wipes ./public and db.json (run after layout/config changes if dev server shows stale output)
npm run update   # bump deps via npm-check-updates
```

There are no tests, linter, or formatter configured.

## Deployment pipeline (`.github/workflows/jekyll-gh-pages.yml`)

The workflow name is misleading ‚Äî this is **not** Jekyll. On every push to `master`, GitHub Actions:

1. `npm install && npm run build` ‚Üí Hexo writes the site to `./public`.
2. Installs Python `rendercv[full]`, runs `rendercv render Vladimir_Babin_CV.yaml` inside `source/cv/`, and copies `rendercv_output/*` plus the source YAML into `public/cv/`. **The CV is rendered at deploy time, not by Hexo.** Editing `source/cv/Vladimir_Babin_CV.yaml` is what changes the published CV.
3. Copies everything from `./resources/` directly into `./public/` (e.g. `resources/virtualKeyboard.html` ships as `babin.info/virtualKeyboard.html`). Use `resources/` for static files that should bypass Hexo rendering entirely.
4. Writes `babin.info` into `public/CNAME` and deploys via `actions/deploy-pages`.

When adding a new top-level static asset, decide between three locations: a Hexo page under `source/`, a passthrough file under `resources/`, or CV-related output under `source/cv/`.

## Content layout

- `source/_posts/` ‚Äî blog posts in Markdown. Filenames may be Cyrillic; the permalink pattern is `post/:title/` (see `_config.yml`).
- `source/index.md`, `source/404/`, `source/cv/`, `source/SymplastFellowshipPrototype/` ‚Äî standalone pages. Anything in `source/` not under `_posts/` becomes a regular page.
- `scaffolds/` ‚Äî templates used by `hexo new <layout> <title>` (`post`, `page`, `draft`).
- `public/` ‚Äî build output, gitignored, regenerated by `npm run build`.
- `db.json` ‚Äî Hexo's local cache, gitignored.

## Notes

- `_config.yml` sets `language: ru`, `permalink: post/:title/`, theme `landscape`, and a sitemap generator. Most site-wide changes belong here rather than in theme files.
- `.nojekyll` exists at the root to tell GitHub Pages not to run Jekyll on the deployed artifact.
